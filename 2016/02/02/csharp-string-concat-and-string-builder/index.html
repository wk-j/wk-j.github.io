+ กับ StringBuilder

2016-02-02

Concat String ด้วยเครื่องหมาย +

เนื่องจาก string เป็น immutable type ทุกครั้งที่ x += "!" ทำงานจะมีการสร้าง string ขึ้นใหม่ โดยคัดลอก string เดิมต่อกับเครื่องหมาย ! ไปเก็บใน memory ที่มีการ allocate ขึ้นใหม่ จากโค้ดข้างบนจะมีการคัดลอก string ถึง 100,000 ครั้ง แต่ละครั้งจะใช้เวลามากขึ้นเรื่อยๆ เนื่องจากจากขนาดของ string ยาวขึ้น

Concat String ด้วย StringBuilder

StringBuilder จะมี buffer อยู่ภายใน เมื่อมีการการต่อ string ด้วยฟังก์ชัน Append จะไม่จำเป็นต้องจอง memory ใหม่ทุกรอบ แต่จะเกิดขึ้นก็ต่อเมื่อ memory หรือ buffer ที่มีอยู่ไม่พอ โดยจะเพิ่มขนาดของ buffer เป็นสองเท่าของขนาดเดิม จากโค้ดทั้งสองแบบจึงเห็นได้ว่า ในกรณีที่มีการต่อ string จำนวนมาก การใช้ StringBuilder จะให้ประสิทธิภาพที่ดีกว่า
